# Contributing

## Before you commit

- Run the main pipeline to ensure it still works:
  ```bash
  python fetch_all_quarterly.py
  ```
  This will fetch data, merge to `latest_quarter_financials.json`, and build `auto_suppliers.xlsx`.

- If you change only the Excel builder (no fetch changes), you can run:
  ```bash
  python build_auto_excel.py
  ```

## Data files

- **ticker_source_map.json** — Company keys must match `build_auto_excel.py` INCLUDE_DETAILS and FINANCIALS. Use exact company names as in the code.
- **latest_quarter_financials.json** — Generated by the pipeline. Consider adding it to `.gitignore` if it is large or you prefer not to commit fetched data; the workbook can be rebuilt from FINANCIALS fallback when the JSON is absent.
- **quarterly_overrides.json** — Optional; only add if you are filling gaps for specific companies.

## Code style

- Use type hints and docstrings (Google-style) for public functions.
- Preserve existing comments when editing; add comments for non-obvious logic.
- Run from project root; for `recalc.py` use `PYTHONPATH=scripts` so `office.soffice` resolves.

## Adding companies

1. Add the company to `build_auto_excel.py`: INCLUDE_DETAILS (tier, parts, oems, duns) and, if needed, FINANCIALS (fallback tuple) and EXCLUDE_REASONS if it is excluded.
2. For US-listed companies, add an entry to `ticker_source_map.json` with `data_source: "US"` and the correct ticker.
3. For Canada/Europe/Korea, add to `ticker_source_map.json` and to `GLOBAL_QUARTERLY` in `fetch_quarterly_global_yf.py` (name, Yahoo symbol, currency).
